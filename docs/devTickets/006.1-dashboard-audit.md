## Key Findings

### Current scaffold (`DashboardPage.jsx`)

| Component | Current content | Data source |
|-----------|-----------------|---------------|
| Interactive CV | Name, headline, skill chips, evidence cards | Hard-coded dummy |
| Evidence Summary | Static placeholder copy | None |

### Target layout (from audit)

The Dashboard should remain a **two-column workspace**, but the **Interactive CV is the hero**, not a collection of disconnected utility widgets.

```
┌─────────────────────────────────────┬──────────────────────────┐
│  Interactive CV (primary)           │  Profile completeness    │
│  - Name + headline                    │  (only when incomplete)  │
│  - Profile summary (CoreContext)      ├──────────────────────────┤
│  - Core competencies (future/derived) │  Collapsible panel:      │
│  - Top experiences/projects (future)│  Evidence Summary   OR   │
│                                       │  Recent Activity         │
└─────────────────────────────────────┴──────────────────────────┘
```

### Interactive CV — target content

Inspired by a real CV structure, the Interactive CV should eventually contain:

| Section | Phase 1 source | Later source |
|---------|------------------|--------------|
| Name + headline | `CoreContext.fullName`, `CoreContext.headline` | Same |
| Contact strip | `CoreContext.mobile`, `CoreContext.location`, `User.email` | Optional website field |
| Profile summary | `CoreContext.rawSummaryMd` (preview) | AI-polished summary (future) |
| Core competencies | Placeholder / empty chips | Derived from Experiences + Activities |
| Experiences & projects | Placeholder cards | Top 2–3 notable `Experience` records |

Phase 1 does **not** require Experience APIs. The CV shows identity + summary + empty-state guidance for evidence sections.

### Evidence Summary vs Recent Activity

Audit decision: **Option 2 — collapsible alternate panels**

- Evidence Summary and Recent Activity share the right column height with the Interactive CV.
- User toggles which panel is expanded.
- Phase 1: both show guided empty states.
- Phase 2+: Evidence Summary shows compact experience highlights; Recent Activity shows top 5 recently updated evidence items (Activities, Journal entries — not limited to the `Activity` entity name).

### Profile completeness

- Shown only when the profile is incomplete (not a permanent fixture when ready).
- Prompts user toward `/profile` with a clear next action.

### Opportunities

- Reserved for a later phase; no Phase 1 dashboard widget beyond optional quick-action link labelled “Coming soon”.
