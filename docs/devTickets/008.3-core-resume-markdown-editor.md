# 008.3 — Core Resume Markdown Editor

## Status

**Planned**

## Type

Frontend / UX

## Phase

## Related Screen

## Depends On

- **008.1** — Reusable `MarkdownEditor` widget

- **008.2** — Documents migration plan (no backend change in this ticket)

## Objective

Replace the plain `TextArea` in `CoreResumeEditor` with the shared `MarkdownEditor` component — matching Core Context raw summary and Journal notes UX.

This is a **UI-only** change. Storage remains `User.coreResumeMd` via existing profile API.

---

## Current State

```
client/src/features/profile/components/CoreResumeEditor.jsx
  └── TextArea (mono, min-h 280px)
```

## Target State

```
client/src/features/profile/components/CoreResumeEditor.jsx
  └── MarkdownEditor
        label="Resume Markdown"
        previewEnabled
        minRows={14}
        helperText aligned with strategic resume copy
```

---

## Technical Tasks

1. Import `MarkdownEditor` from `client/src/components/editor/MarkdownEditor.jsx`.
2. Remove `TextArea` usage for resume body.
3. Wire controlled `value={coreResumeMd}` and `onChange` (string) — same pattern as `CoreContextEditor`.
4. Keep existing Card layout, save button, save status, and error alert unchanged.
5. Adjust placeholder and helper text for resume-specific guidance (headings, experience bullets).
6. Optional: `minRows={14}` or similar for resume-length content.
