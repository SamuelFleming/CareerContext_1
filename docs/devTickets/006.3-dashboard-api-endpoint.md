## Objective

Implement `GET /api/dashboard` — an authenticated endpoint returning the reconciled dashboard payload defined in **006.2**.

Replace the current 501 scaffold in `dashboardController.js`.

## Endpoint

```
GET /api/dashboard
```

| | |
|---|---|
| Auth | Required |
| Success | `200` with `{ data: ... }` |
| Unauthenticated | `401` |

---

## Technical Tasks

Create or update:

```
server/src/services/dashboardService.js
server/src/controllers/dashboardController.js
server/src/routes/dashboardRoutes.js
```

### Service responsibilities

1. Load authenticated `User` by JWT `userId`.
2. Load or create `CoreContext` via `coreContextService.ensureForUser`.
3. **Calculate `profileCompleteness`** per **006.2** weighting rules.
4. **Build `identity`** from `CoreContext` + `User.email`.
5. **Build `interactiveCv`** summary preview (250–400 chars), staleness flag, empty arrays for Phase 1 competencies/experiences.
6. **Build `coreResumePreview`** from `User.coreResumeMd` / `coreResumeUpdatedAt`.
7. **Build `evidencePanel`** with Phase 1 placeholder payloads and zero counts.
8. **Build `quickActions`** array per contract.
9. Return `{ data }` envelope.

### Suggested helpers (internal to service)

```
calculateProfileCompleteness(user, coreContext) → profileCompleteness
buildSummaryPreview(rawSummaryMd, maxLength = 350) → string | null
isReviewSuggested(summaryUpdatedAt, days = 30) → boolean
buildResumePreview(coreResumeMd, maxLength = 200) → { exists, previewMd, updatedAt }
```

### Route

- Apply `authenticateWithJwt` on `GET /`

---

## Response Shape

Must match **006.2** contract exactly (keys and semantics).

Do **not** return `passwordHash`.

---

## Out of Scope

- Experience, Activity, Journal, Opportunity, or Document queries
- Populating `coreCompetencies`, `highlightExperiences`, or `recentActivity.items`
- Quick-action route existence validation

## Acceptance Criteria

- [ ] `GET /api/dashboard` requires authentication
- [ ] Unauthenticated request returns `401`
- [ ] Authenticated user receives full `data` payload
- [ ] `identity` uses `CoreContext.fullName`, `CoreContext.headline`, `User.email`
- [ ] `profileCompleteness.score` matches documented weighting
- [ ] `profileCompleteness.showPrompt` is `false` when `status === "ready"`
- [ ] `interactiveCv.summaryPreview` respects length and empty rules
- [ ] `interactiveCv.reviewSuggested` applies 30-day staleness rule
- [ ] `coreResumePreview` reflects resume presence and preview
- [ ] `evidencePanel` returns Phase 1 placeholders with zero counts
- [ ] `quickActions` includes enabled Edit Core Context + disabled future actions
- [ ] Existing auth and profile endpoints unaffected
- [ ] `server/README.md` endpoint log updated
