```
GET /api/dashboard
```

| | |
|---|---|
| Auth | Required |
| Success | `200` with `{ data: ... }` |
| Unauthenticated | `401` |

## Implementation

```
server/src/services/dashboardService.js
server/src/services/profileCompletenessService.js   # shared completeness rules
server/src/constants/phase1DashboardMocks.js        # Phase 1 CV mock data
server/src/controllers/dashboardController.js
server/src/routes/dashboardRoutes.js
```

### Service responsibilities

1. Load authenticated `User` by JWT `userId`.
2. Load or create `CoreContext` via `coreContextService.ensureForUser`.
3. Calculate `profileCompleteness` via `profileCompletenessService` (shared with `GET /api/profile`).
4. Build `identity` from `CoreContext` + `User.email`.
5. Build `interactiveCv` summary preview (max 350 chars), staleness flag, Phase 1 mock competencies and experience highlights.
6. Build `evidencePanel` with Phase 1 placeholder payloads and zero counts.
7. Return `{ data }` envelope.

### Not included (removed from scope)

- `coreResumePreview`
- `quickActions`
