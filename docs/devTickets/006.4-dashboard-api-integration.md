### `DashboardPage.jsx`

- Fetch dashboard on mount
- Loading state (skeleton or simple message)
- Error state with retry
- Two-column grid matching current layout (`lg:grid-cols-[1.4fr_1fr]`)

### Component mapping

| API key | Component | Notes |
|---------|-----------|-------|
| `identity` + `interactiveCv` | `InteractiveCvCard` | Name, headline, contact strip, summary preview, competency chips (empty state), experience highlight area (empty state) |
| `profileCompleteness` | `ProfileCompletenessPrompt` | Render only when `showPrompt === true` |
| `evidencePanel` | `EvidencePanel` | Collapsible toggle: Evidence Summary ↔ Recent Activity |
| `coreResumePreview` | `CoreResumePreviewCard` | Exists/preview/link to `/profile` |
| `quickActions` | `QuickActionsCard` | Enabled/disabled actions with badges |

### Collapsible evidence panel (006.1 decision)

- Single card with tab/toggle controls
- Views: `evidenceSummary` | `recentActivity`
- Phase 1: render `message` from API for each view
- Preserve card height alignment with Interactive CV column where practical

### Styling

- Reuse existing `Card`, `PageHeader`, `SkillChip`, `EvidenceCard` components
- Match tokens and spacing from current `DashboardPage`
- Do not regress the Interactive CV aesthetic

## UX Behaviour

| State | Behaviour |
|-------|-----------|
| Loading | Show loading indicator; no dummy names |
| Error | Alert + “Try again” button |
| Empty summary | Guided copy inside CV profile section |
| Empty competencies | Placeholder chips or helper text |
| Empty experiences | Single empty-state line inside CV |
| Profile ready | Hide completeness prompt |
| Future quick actions | Visually disabled with “Coming soon” badge |

---

## Out of Scope

- Wiring Experience or Journal APIs
- Populating real skill chips or evidence cards
- Opportunities widget
- Dashboard edit interactions (read-only preview)
