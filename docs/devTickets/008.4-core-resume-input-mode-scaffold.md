# 008.4 — Core Resume Input Mode Scaffold (Markdown vs Upload)

## Status

**Planned**

## Type

Frontend / UX scaffold

## Phase

Phase 1 — Auth and Profile (UI only)

## Related Screen

```
5. Profile / Core Context
```

## Depends On

- **008.3** — `MarkdownEditor` in `CoreResumeEditor`
- **008.2** — Documents migration plan (defines why upload is not persisted yet)

## Objective

Scaffold a **dual input mode** on the Core Resume section so users can see the future choice between native Markdown editing and file upload — without implementing Document backend, file storage, or text extraction.

---

## UX Design

### Mode selector

Segmented control or tab pair at the top of the Core Resume card body:

| Mode | Label | Phase 1 behaviour |
|------|-------|-------------------|
| `markdown` | Write in Markdown | Active default — `MarkdownEditor` from **008.3** |
| `upload` | Upload file | Scaffold panel only — no persistence |

### Upload scaffold panel

When `upload` is selected, show a non-functional upload area:

- Dashed border dropzone styling (matches Profile evidence/warm surfaces)
- Copy: e.g. “Upload PDF or DOCX — available when Document storage lands (Phase 3)”
- Optional disabled “Choose file” button or visibly disabled dropzone
- Short note linking to **008.2**: files will be stored as `Document` with `filePath` and imported into Markdown `content`

**Do not:**

- Call `/api/documents`
- Use Multer or client-side file upload to server
- Store files in `localStorage` long-term
- Change `coreResumeMd` from an upload action

### State

- `inputMode`: `'markdown' | 'upload'` — **local React state only** (session UI preference; optional `localStorage` key `coreResumeInputMode` if desired — not required for MVP scaffold)
- Saving resume still only applies in **markdown** mode via existing API

## Suggested Component Structure

Refactor `CoreResumeEditor.jsx` or extract:

```
client/src/features/profile/components/CoreResumeEditor.jsx
  ├── Card header (unchanged — title, save)
  ├── CoreResumeInputModeToggle     # new — markdown | upload
  ├── MarkdownEditor                # when mode === 'markdown'
  └── CoreResumeUploadPlaceholder   # when mode === 'upload' — new small component
```

Optional shared UI:

```
client/src/components/ui/SegmentedControl.jsx   # only if reusable pattern emerges
```

Keep scope minimal — inline toggle in `CoreResumeEditor` is acceptable.

---

## Visual / Copy Guidelines

- Match Profile card tokens (`--evidence-warm`, dashed borders like Journal destination placeholder)
- Upload panel should feel intentional, not broken — clear “coming soon” messaging
- Default mode on load: **markdown**
- If user switches to upload, hide `MarkdownEditor` but do not clear `coreResumeMd`
